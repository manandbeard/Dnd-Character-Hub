# AI-Enhanced D&D Character Builder, Sheet, and Party Ledger – Feature Ideas and Implementation Plan

## Executive overview

Modern D&D tools combine digital character sheets, automated rules calculations, campaign management, and shared party inventory, with newer products layering AI for backstory generation, rules lookup, and creative assistance.[cite:1][cite:5][cite:7][cite:19][cite:28] This report distills patterns from popular apps (D&D Beyond, Roll20, Foundry modules, Party Loot, AI character generators) into concrete feature ideas and an implementation plan for a full stack app that uses your local JSON rules data and supports shared party ledgers and social/campaign connections.[cite:14][cite:21][cite:8][cite:22][cite:29]

## Landscape scan: existing tools and patterns

### Digital character sheets and builders

Several mobile and web apps provide full 5e character sheets with automation for stats, inventory, spellcasting, and leveling.[cite:1][cite:4][cite:15] Typical features include:

- Multi-page character sheets that automatically calculate ability modifiers, skill bonuses, armor class, and attack/damage values.[cite:4][cite:15]
- Spellbooks with slots tracking, save DC, attack bonuses, and preparation management.[cite:1][cite:4][cite:7]
- Inventory pages with custom items, weight tracking, currency tracking, and notes/feature pages.[cite:1][cite:4]
- Character creators that guide new players step-by-step through race, class, background, and starting equipment, often enforcing SRD rules and helping with level-up choices.[cite:1][cite:4][cite:7]

Online platforms like Roll20 and D&D Beyond extend this by tightly coupling sheets with VTT or platform features, such as integrated dice rollers, real-time updates, and one-click export/import between builder and play surface.[cite:7][cite:15]

### Shared party inventory and ledgers

D&D Beyond recently introduced a Party Inventory feature that gives each campaign a shared stash alongside individual character inventories.[cite:5][cite:14] Key patterns:

- A dedicated "Party Inventory" section parallel to "My Inventory" on character sheets.
- Items and currency can be moved between a character’s inventory and the shared party stash, with ownership displayed and synchronized for all players in the campaign.[cite:5][cite:14]
- Party currency pooling with support for all coin types, plus containers that group chunks of currency for easier spending.[cite:5]

Community-built tools show additional ledger patterns. A popular Excel-based "Party Ledger" template tracks coin distribution, shared funds, debts, and loans, calculates coin weight, and keeps multiple sheets for up to seven party members sharing a fund.[cite:2] Web tools like Party Loot offer shared party funds, item ownership tracking, transaction history, and mobile-friendly access, often relying on links for read-only sharing with the party.[cite:8]

Foundry VTT modules such as Party Inventory and Party Sheet provide party-level views that aggregate items flagged as party loot across character sheets or list actors’ key stats, with robust distribution and splitting features.[cite:21][cite:27] These typically:

- Add UI toggles on items to mark them as "Party Loot" so they appear in a centralized party inventory window while still belonging to individual characters.[cite:21]
- Provide scratchpads for staging loot then distributing to characters, splitting stacks, and showing live remainders during distribution.[cite:21]
- Offer party overview sheets showing stats and conditions for all actors in a campaign.[cite:27]

### Campaign management and collaboration tools

Campaign management apps focus on notes, NPCs, locations, timelines, and cross-linking rather than core character math.[cite:20][cite:29] Tools like Goblin’s Notebook, Campaign Logger, Notion, Obsidian, LoreKeeper, and Archivist emphasize:

- Wiki-like pages, interactive maps, and timeline views with heavy cross-linking between characters, locations, items, and sessions.[cite:20][cite:29]
- Real-time session logging, tagging, and campaign logging modes that record changes and notes as you play.[cite:20][cite:26]
- Shared hubs for the whole party and DM, with collaborative note-taking, AI-assisted recaps, and centralized campaign memory.[cite:26][cite:29]

Some newer tools also position themselves as social/campaign discovery platforms, connecting D&D fans based on playstyle and preferences while planning to add campaign notes and scheduling features.[cite:17]

### AI-enhanced character tools

AI-powered character creators focus on narrative content and art generation layered on top of structured stat blocks.[cite:16][cite:19][cite:22][cite:28] Common AI features include:

- Backstory generation (300+ word narratives) tuned by race, class, alignment, and high-level themes such as revenge arcs, tragic pasts, or comic relief.[cite:19][cite:22][cite:28]
- Personality scaffolds (traits, ideals, bonds, flaws) to make characters feel distinct without requiring the player to write everything from scratch.[cite:19][cite:22]
- Character portrait generation (Stable Diffusion or similar models) tailored to user-selected race, class, and aesthetic preferences.[cite:16][cite:19][cite:25][cite:28]
- Idea generation for character concepts, descriptions, and refinements in an interactive assistant workflow.[cite:22][cite:28]

## Feature ideas for your app

### Core character builder and sheet

Based on existing tools, a rich builder in your app should support:

- Stepwise character creation wizard (Race → Class → Background → Abilities → Skills → Equipment → Spells), grounded in your JSON SRD-like data.
- Automatic rules calculations: ability modifiers, proficiency bonuses, skill bonuses, saving throws, attack and damage rolls, spell DC/attack, passive scores, and encumbrance.[cite:4][cite:15]
- Multi-tab sheet UX: Overview (HP, key stats, actions), Abilities/Skills, Combat (attacks, conditions), Spells, Inventory/Currency, Features & Traits, Notes.
- Level-up assistant that walks players through hit points, new features, spell selections, and ASI/feat choices in order.[cite:1][cite:4]
- Offline-friendly local persistence with cloud sync for multi-device players.

### Party and campaign layer

To enable campaign-level collaboration:

- Campaign entity: A campaign with metadata (name, DM, system/version, schedule, description) to which players attach characters.
- Party view: A campaign-level dashboard showing all party members with key stats (HP, AC, passive Perception, conditions) plus quick links to their sheets, similar to Foundry’s Party Sheet.[cite:27]
- Session notes: Campaign-specific notes area with tagging for NPCs, locations, and items, plus optional AI-assisted recaps after sessions.[cite:20][cite:26][cite:29]
- Party chat: In-app text channels at the campaign level for between-session planning, with structured channels (general, loot, downtime, scheduling).

### Shared Party Ledger and inventory

Building on D&D Beyond, Party Loot, and Foundry modules, a "Shared Party Ledger" in your app can:

- Maintain a party-level inventory that is distinct from but linked to individual character inventories, with a clear ownership field.
- Provide a party currency pool with automatic conversion between denominations and optional containers (e.g., labeled pouches of 100 gp).[cite:5][cite:8]
- Track all loot transactions (source, date, who added/removed items, distribution events) as a ledger so players and DM can audit history.[cite:2][cite:8]
- Support splitting stacks and auto-distributing items/coins evenly across the party, calculating remainders.[cite:2][cite:21]
- Handle debts/loans between characters and shared party fund contributions.
- Offer a "party stash" scratchpad for newly found loot during sessions, which can be resolved into individual or party ownership at the end of the session.[cite:21]

### Social and connection features

To help players connect around campaigns:

- Player profiles: Basic profiles with time zone, experience level, playstyle tags (roleplay-heavy, tactical, rules-light), and availability windows.
- Campaign recruitment: DM-created listings with filters for system, level range, and schedule; players can request to join.
- Safety and moderation primitives: Invite codes, soft-blocks, and reporting tools, plus the ability to make campaigns private or invite-only.
- Optional integrations (later phases): External identity linking (Discord, Google), calendar export for session times.

### AI capabilities tailored to your stack

Given access to your JSON rules data and campaign/character context, AI can assist in ways that go beyond generic character generators:

- Backstory generator that reads selected race, class, background, alignment, and campaign tone to generate a tailored backstory and personality scaffold, similar to BasedLabs and other AI generators.[cite:19][cite:22][cite:28]
- Character refinement coach: An in-app chat mode tied to the character that suggests feats, spells, and equipment based on build goals (e.g., "frontline tank," "battlefield control caster") using your data.
- Session recap agent: Given structured session notes or chat logs, generate in-character or neutral recaps, highlight unresolved hooks, and update campaign tags.[cite:26][cite:29]
- Rules explainer: Context-aware rules lookup and explanation using your JSON rules, answering questions like "How does this class feature interact with this spell?" with references to source sections.
- Party ledger assistant: Smart suggestions about splitting treasure, reminding players about encumbrance, and highlighting under-geared characters based on comparison to party averages.[cite:2][cite:21]
- Art hooks: Integrations or prompt templates for external AI art tools (Hotpot, Stable Diffusion, etc.) to generate character portraits and key campaign images.[cite:16][cite:25][cite:28]

## High-level architecture

### Domain model

At a minimum, the domain model should include:

- User: Authentication identity, profile data, preferences.
- Campaign: Metadata, DM, members, sessions, notes, tags, privacy settings.
- Character: Core stats, derived stats, class levels, race, background, abilities, skills, inventory, spells, notes, owner, campaign link.
- PartyInventory: Linked to campaign; contains Items and CurrencyEntries.
- Item: From your JSON rules or homebrew; attributes for type, rarity, weight, attunement, etc., plus owner (character or party) and history.
- CurrencyEntry: Denomination, amount, owning entity (character vs party), and container label.
- LedgerEntry: Records a transaction (type, timestamp, source, target, items/currency involved, notes).
- Session: Date, title, summary, raw notes, recap, and associated changes (loot, XP, etc.).
- AIJob: For asynchronous AI tasks like backstory generation or long recaps; stores input context and output.

Your existing JSON rules data for classes, races, items, and spells can map into normalized tables/collections or be loaded as reference documents for rules calculations.

### Backend services

A typical backend stack could include:

- REST or GraphQL API for client communication.
- Authentication and authorization layer with JWTs or session tokens; campaign- and character-level ACLs.
- Business logic services for character creation, level-up, inventory transfers, ledger operations, and AI orchestration.
- Real-time subsystem (WebSockets or WebRTC-based signaling) for live campaign updates, chat, and shared ledger updates.
- Background worker for AI jobs and periodic maintenance (e.g., sending recap emails, cleaning old logs).

The AI layer can be abstracted as a service with endpoints like `POST /ai/characters/backstory`, `POST /ai/campaigns/recap`, and `POST /ai/ledger/split`, each reading domain context from the database and combining it with prompts to your LLM provider.

### Frontend application

On the frontend, align UX patterns with existing successful apps:

- Character-centric views: A primary character sheet layout with responsive tabs and prominent actions (rolls, rests, level up, edit notes).[cite:4][cite:7][cite:15]
- Campaign dashboard: Cards for each campaign showing upcoming session times, last recap, party status, and quick links.
- Party inventory panel: A dedicated tab or modal exposing party stash, with drag-and-drop between characters and party plus ledger history.[cite:5][cite:8][cite:21]
- Builder wizard: A modal or multi-step page guiding new characters with progress indicators and inline explanations of choices.[cite:1][cite:4][cite:7]
- AI sidebars: Contextual side panels for "Ask AI about this character" or "Generate recap" so AI feels additive, not intrusive.

## Implementation phases

### Phase 1: Foundation and local rules integration

Goals for this phase:

- Import and normalize your existing JSON data for races, classes, items, spells, and backgrounds.
- Implement user accounts and basic auth.
- Build a minimal character schema and read-only character sheet UI with derived stat calculations.
- Implement character creation wizard (core SRD options) using your JSON data for choices and rules.

Key tasks:

1. **Data layer**
   - Define schemas/tables for RulesReference entities and write an importer that ingests your JSON files.
   - Implement a rules engine module that, given a character build, computes derived stats and valid options for level-up.

2. **Auth and user model**
   - Implement signup/login, password reset, and session management.
   - Define `User` model with basic preferences (theme, time zone, system version).

3. **Character core**
   - CRUD endpoints for characters with validation against rules data.
   - Character sheet UI (stats, skills, HP, AC, inventory skeleton, features list) in a web frontend.

### Phase 2: Campaigns and shared party ledger

Goals for this phase:

- Introduce Campaigns and attach characters to campaigns.
- Implement the Party view and Shared Party Ledger with basic inventory and currency functionality.

Key tasks:

1. **Campaigns and membership**
   - Create Campaign model with owner (DM), name, description, system, and privacy.
   - Implement invitations (invite code or email link) and membership roles (DM, player).

2. **Party dashboard**
   - Build a campaign overview page showing characters, key stats, and basic notes.[cite:27]
   - Add party chat (either internal or via integration later).

3. **Party ledger and inventory**
   - Implement PartyInventory and LedgerEntry models.
   - Add UI to mark items as party loot and toggle visibility in the party inventory.[cite:5][cite:21]
   - Create a Party Inventory screen with:
     - Party currency pool and currency containers.[cite:5][cite:8]
     - Item list with ownership and drag-and-drop to move between party and characters.
     - Transaction history and filters.
   - Implement coin split/distribution helpers mirroring spreadsheet-based party ledgers.[cite:2][cite:21]

### Phase 3: AI-enhanced workflows

Goals for this phase:

- Add AI features that leverage existing data without destabilizing core flows.

Key tasks:

1. **AI backstory and personality generator**
   - Build an AI service endpoint that, given a character’s race, class, background, alignment, and player prompts, returns a backstory and personality traits.[cite:19][cite:22][cite:28]
   - Add UI in the character creator for "Generate backstory" and "Regenerate / refine".

2. **Rules and build assistant**
   - Implement a context-aware AI helper that can answer rules questions based on your JSON reference.
   - Add a build advisor that suggests feats, spells, and items aligned with selected playstyle tags.

3. **Session recap assistant**
   - Extend Campaign sessions with raw notes or chat logs.
   - Add an AI endpoint to produce recaps and highlight unresolved threads.[cite:26][cite:29]

4. **Party ledger assistant**
   - Add AI helper for treasure division, debt tracking, and identifying under-geared party members based on grouped stats.[cite:2][cite:21]

### Phase 4: Social graph and discovery

Goals for this phase:

- Enable players to discover campaigns and connect with others, inspired by social campaign management tools.[cite:17][cite:29]

Key tasks:

1. **Player profiles and preferences**
   - Extend User model with playstyle tags, preferred systems, availability, and experience level.

2. **Campaign discovery**
   - Create public campaign listing endpoints and filters.
   - Implement join requests and DM approval flows.

3. **Safety and moderation**
   - Add basic blocking, reporting, and campaign privacy controls.

4. **AI-enhanced matching** (optional)
   - Use AI to suggest compatible players/campaigns based on profile and behavior.

## Development instructions for a Vibe Code Agent

The following instructions can be given directly to an autonomous coding agent to implement the app incrementally.

### General guidelines

- Use a modern full stack framework (e.g., Next.js/React on the frontend, Node/TypeScript backend or a unified full stack framework) with a strongly typed domain model.
- Treat the local D&D JSON data as canonical rules reference; create importers and typed wrappers rather than scattering JSON access throughout the codebase.
- Prioritize deterministic rules calculations and data integrity before adding AI.
- Design for multi-tenant campaigns from day one; all characters, sessions, and ledgers should be scoped to a campaign when applicable.

### Step-by-step build order

1. **Set up project and core domain**
   - Initialize repository, CI, linting, and testing.
   - Define core TypeScript types for User, Character, Campaign, Item, PartyInventory, CurrencyEntry, LedgerEntry, and Session.
   - Implement database schemas and migrations for these entities.

2. **Rules data ingestion**
   - Build a CLI or script that loads the provided JSON files (races, classes, items, spells, backgrounds) into a RulesReference store (database tables or JSON documents).
   - Implement a RulesService that exposes high-level queries such as `getClassFeatures(level)`, `getSpellListForClass`, `getStartingEquipment`, and `computeAbilityModifiers`.

3. **Character sheet and builder**
   - Create backend endpoints for character CRUD with validation using RulesService.
   - Implement a multi-step character builder UI that uses rules data to present legal choices and updates a preview pane.
   - Implement a character sheet UI with derived stats and basic editing (notes, inventory descriptions).

4. **Campaigns and membership**
   - Implement Campaign model, membership roles, and permissions middleware.
   - Create API endpoints to create, update, and list campaigns and to attach characters to campaigns.
   - Build a campaign dashboard UI showing members and quick stats.

5. **Shared Party Ledger and inventory**
   - Implement PartyInventory and LedgerEntry tables linked to Campaign.
   - Add APIs for:
     - Marking an item as party loot / removing it from party loot.
     - Moving items and currency between characters and party.
     - Creating ledger entries for each transaction.
   - Build frontend components for Party Inventory:
     - List of party items with owners, filters, and search.
     - Party currency pool with containers and distribution actions.
     - Transaction history with timestamps and actors.
   - Add helper functions for equal coin distribution, stack splitting, and remainder handling.

6. **Real-time updates and chat**
   - Add WebSocket or similar real-time layer.
   - Implement campaign-scoped chat channels and live updates for Party Inventory changes.

7. **AI integration**
   - Abstract AI provider behind an `AiService` that accepts structured prompts and context (character snapshot, campaign notes).
   - Implement endpoints for:
     - Character backstory generation.
     - Session recap generation.
     - Build and rules assistance (Q&A).
     - Party ledger advice.
   - Integrate AI actions into the UI as optional helpers, ensuring core flows work without AI responses.

8. **Social and discovery features**
   - Extend User model with profile information and preferences.
   - Implement public campaign listings and filters with privacy controls.
   - Add join request and approval flows.

9. **Polish, accessibility, and DM tools**
   - Refine UX for keyboard navigation, screen reader support, and responsive layouts.
   - Add DM-facing tools like quick party overview, encounter notes, and per-session loot staging.
   - Implement export/import for characters and campaigns for backup and portability.

By following these phases and instructions, the agent can build a robust, AI-augmented D&D character builder and campaign tool that aligns with modern ecosystem patterns while leveraging your JSON rules data and emphasizing collaborative features like a Shared Party Ledger.[cite:1][cite:2][cite:4][cite:5][cite:7][cite:8][cite:14][cite:19][cite:21][cite:26][cite:27][cite:29]
